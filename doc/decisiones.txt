2. Acciones del cliente en _procesar_notificacion
──────────────────────────────────────────────────────
Las keys vienen de config (aceptar, rechazar, esperar, entendido, consultar),
pero los side-effects son distintos:
  - aceptar / rechazar → cambian estado_item + evalúan
  - esperar            → agenda recordatorio (placeholder)
  - entendido          → solo marca leído
  - consultar          → entra al sub-flujo escribir_consulta

Puedo mover estado_item_destino y texto_accion al JSON, pero los efectos
especiales (esperar, consultar) seguirían siendo lógica Python.

¿Solo movemos los datos (estado destino, texto)?
¿O también definimos un mecanismo de accion_especial en config?

3. Destinos de transición desde el lado del cliente
──────────────────────────────────────────────────────

Cuando el cliente envía token    → token_enviado (hardcodeado)
